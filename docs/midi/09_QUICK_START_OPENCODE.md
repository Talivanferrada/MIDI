# 09 — Quick Start para OpenCode/GSD

## Opción recomendada

En una sesión nueva, no pegues todo el contexto en el chat.

Usa estos archivos como contexto.

## Paso 1

Pide al agente:

```text
Lee `docs/midi/00_PROJECT_BRIEF.md` a `docs/midi/05_OUTPUTS.md` como fuente de verdad del proyecto MIDI.

Luego ejecuta `docs/midi/06_PHASE_1_PROMPT.md`.

No simplifiques el sistema a un solo prompt. Crea el repositorio instalable real.
```

## Paso 2

Cuando Phase 1 termine:

```text
Lee nuevamente `docs/midi/00_PROJECT_BRIEF.md` a `docs/midi/05_OUTPUTS.md`.

Ahora ejecuta `docs/midi/07_PHASE_2_PROMPT.md`.

Respeta la arquitectura existente y desarrolla el núcleo real de MIDI.
```

## Paso 3

Cuando Phase 2 termine:

```text
Lee nuevamente `docs/midi/00_PROJECT_BRIEF.md` a `docs/midi/05_OUTPUTS.md`.

Ahora ejecuta `docs/midi/08_PHASE_3_PROMPT.md`.

Valida, rompe, corrige y endurece MIDI.
```

## Prompt corto alternativo

```text
Estás construyendo MIDI — Modelo Inteligente de Desarrollo de Innovación.

MIDI es un framework multiagente instalable, inspirado en GSD/SDD, que transforma ideas iniciales en proyectos completos, analizados, tensionados, validados y adaptables a fondos, inversionistas o ejecución real.

Lee los archivos `docs/midi/` como fuente de verdad y ejecuta la fase que te indique.
```
