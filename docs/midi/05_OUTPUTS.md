# 05 — Outputs MIDI

## Archivos de estado

### PROJECT_STATE.md

Debe contener:

- nombre del proyecto,
- idea inicial,
- modo activo,
- etapa actual,
- idea seleccionada,
- top 3,
- hipótesis principales,
- riesgos críticos,
- análisis pendientes,
- próximos pasos,
- fecha de actualización.

### DECISION_LOG.md

Debe registrar:

- fecha,
- decisión,
- agente,
- motivo,
- alternativas descartadas,
- supuestos,
- impacto.

### ASSUMPTIONS.md

Debe registrar:

- supuesto,
- nivel de confianza,
- impacto,
- cómo validarlo,
- estado.

### RISK_REGISTER.md

Debe registrar:

- riesgo,
- categoría,
- probabilidad,
- impacto,
- severidad,
- mitigación,
- agente que lo detectó,
- estado.

### USER_CONTEXT.md

Debe contener:

- país,
- región,
- ciudad/comuna,
- presupuesto,
- habilidades,
- recursos,
- red de contactos,
- intereses,
- restricciones,
- riesgo aceptado,
- objetivos,
- foco comercial,
- foco fondos,
- tiempo disponible.

### IDEA_BACKLOG.md

Debe contener ideas generadas con:

- nombre,
- descripción,
- cliente,
- problema,
- solución,
- diferenciación,
- innovación,
- dificultad,
- modelo de ingresos,
- riesgos.

### TOP3_IDEAS.md

Debe contener:

- idea 1,
- idea 2,
- idea 3,
- scoring,
- pros,
- contras,
- recomendación.

## Evaluación tipo jurado

Debe incluir:

- puntaje total 0-100,
- puntaje por dimensión,
- fortalezas,
- debilidades,
- condiciones para mejorar,
- recomendación:
  - postular,
  - iterar,
  - validar antes,
  - descartar,
  - pivotar.

## Pitch

Debe incluir:

- problema,
- solución,
- mercado,
- innovación,
- tracción o validación,
- modelo de negocio,
- equipo,
- solicitud o necesidad,
- cierre.
